Data types

Data types are basic building blocks for your data structures and tables. You use them to specify the type of the columns in the tables you create. Data types can be of seven types:

Text datatypes - These values store character strings, such as names, adresses, descriptions, etc...
Numeric datatypes - These values can store numbers, integer or float alike.
Boolean datatypes - The values TRUE or FALSE
Date/Time datatypes - These values will store fo example a date, or a timestamp
Blod datatypes - These values store large binary data, such as the content of a file, or anything else large enough
Miscellaneous datatypes - These values can be of type enumeration or set. You can specify which values to choose from
Spatial datatypes - Used to represent geometrical notions

When you have specified the type of the data type you have to choose the underlying SQL which will be used when creating the actual column in the table. For example, if you chose Text you wil be able to specify it to be CHAR, VARCHAR, etc... for the Numeric type you can choose between INT, FLOAT, etc...

Properties of the data type.

Basic properties

Name: Give a unique name for the data type. A small advice is to follow some consistent notation among your datatypes.

Type: A datatype can be either a number, or a string, a boolean, a date or a.blob. Specify here the type of the datatype

Underlying SQL: Select the SQL type this datatype will have. This combobox is populated bassed on the type of the datatype

Width: This is the width of the data-type. For text/blob type DTs this.is the length of the text, for numerical types this is the max number of column digits. Also you can specify a decimal counter, seprated by a comma from the width (such as 10,2). This field is not used for the other types.

Values: In case you want to create a “Misc” datatype which can choose its values from a list (such as ENUM, or SET) here you can add the initial values of the datatype.

Advanced

Default Value: gives the possibility to specify a default value for the column(s) which are based on this datatype when creating the table.

The "Null" checkbox: check this if you want the NULL to be the default value for th column.

Character set:specify the character set of the data type if using special languages

Description

Provide a description of the data type. It will be used later in the code generation and the document generation phase.

The Importance of Datatypes in database validation

When DDM is performing a validation it is doing a database normalization check (ie. to not to have the possibilty to introduce duplicate data in the tables which will need further maintenance efforts). If DDM finds that two columns in two different tables are having the same Text or Blob data type it considers this is being treated as a normalization issue (since entering data in the same type of fields might lead to data duplication) so it is reported in the issues found. Validations are performed every time a column changes, or on request by using the Validate Solution button in the toolbar.
