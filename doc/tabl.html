Tables

In a relational database a table is a set of data, organized in a model using horizontal rows and vertical columns. The columns are identified by a name, and the rows are identified by the values appearing in the columns belonging to the rows key elements. The table can have a specific number of columns and an unlimited number of rows. DDM allows you to create tables using the Data types yo have defined acting as the types for columns, and also allows you to set up relationships between various tables, which can be later presented in a diagram.

Table Name:Give a unique name for the table type. A small advice is to follow some consistent notation among your tables.
Save: When you're done with the modifications of the current table please press this button.

Columns page

Columns list

The list of columns contains all the columns you have created for this table. The PK field has a icon in case this is a primary key. By clicking on a column in the list you select the column, and you can modify the name, type or other properties of it.

By clicking the button you move the selected column "Up" in the list(ie. to the left in the tables columns), and by clicking the button you move the selected column "Down" (ie. to the rght in the tables columns). By clicking the button you delete the selected column from the list (and from the table). In case this is a specialized table, the columns from the parent table will be shown with a light gray background and you cannot modify or delete them, nor can you change their position.

New Column

Name: Give a unique name in this table for the new column.

Type: Choose the type of the column from the list of Data types you have created.

Primary key: If you want your column to take part from a list of primary key columns, check this checkbox.

Auto increment: The incrementation of Numeric types can be taken care automatically by the MySQL database, however this is ignored for non numeric types.

Description: Give a meaningful description for the column, as this text will appear in the generated code in form of comments and in the generated documentation too.

When you have filled up the required fields to create a column press the button to add the new column to the table. If you have a column already selected the button will be replaced by the button and pressing it will update the column in the table. By pressing the button you will reset the changes you have done and can start over again. No changes will be saved to the selected column.

Indices Page

Indices list

This list contains the indices that were created for the table. By clicking on an index you select it, and you can modify its properties. You also can delete the selected index with the button. The indices starting with "autodix_" are indices that were created because of a foreign key referencing this table and should not be deleted nor modified. These indices are having a dark grey background. In case this table is a spcialized table, the indices from the parent table are presented on a light gray background.

New Index group.

Name: is the name of the index.

Type: you can define the type of the index depending on the Storage Engine and the capabilities of the database.

Columns: the available columns for the index are on the left side, you can use the button to move the selected columnto the right (and you can use the button to remove the selected column from the index). Once a column has been moved to the right side (meaning it was added to the index) you can move them up/down with the / buttons.

When done press the button to add the new index to the table. Pressing the button you will cancel editing the current index.

Foreign keys Page

In case the selected storage engine for the table does not support foreign keys you will see a warning regarding this. For MySQL right now the InnoDB storage engine has support for foreign keys. In case the selected storage engine does not support foreign keys, you still will be able to specify them, and they also will be shown on the E-R diagrams.

Foreign keys list

This list contains the foreign keys that were defined for this table. By clicking on a foreign key you select it and you can delete the selected foreign key with the c button. Foreign key propagation is by default not enabled, meaning, a foreign key defined in a parent table usually is not taken over in the specialized table.


For New Foreign Key

FK Name: the name of the foreign key you will define. This must be unique.



Values:In case you want to create a “Misc” datatype which can choose its values from a list (such as ENUM, or SET) here you can add the initial values of the datatype.
