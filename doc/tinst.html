Table instances

When working in an object oriented modelling project, the table instances are being created from a table template:
- in the project tree right click on the table template, choose Instantiate, or
- Click the "New Table Instance" tool button and in the new dialog box check all the table instance you need
- or Long click the "New Table Instance" button and from the popup select one table instance.

The table instance screen can be used to create the startup values of the tables (ie, values that will be inserted in the tables right after the creation. There is the possibility to import values from CSV files, and also you can save the values for further reference into CSV files. These values will be appended in the table generation SQL script in form of INSERT INTO ... lines of code. In case your table contains an AUTO_INCREMENT column (MySQL) the GUI will allow you to specify the values, however the generated SQL script will NOT contain them to protect the integrity of the data.

With the button you insert a new row in the table, with the button you remove the currently selected row from the table. When you are done with inserting the data press the button to validate the data that was entered by you, and pressthe "Save" button to save the data. Basic validation is done at this, there are some basic error checkings, such as do not enter a string value for a numeric field, but the more complex validations are reserverd for future releases of the application.

When you are instantiating a table that has a foreign key reference to another table the other table will be instantiated also (I call this phenomenon: one table pulled in another table in the instantiation), and in the project tree it will get a specific marking: ie. the icon of the table instance () with a small black dot. When you delete a table instance which pulled in other tables being instantiated will also delete the pulled in tables, and you cannot delete a pulled in table without deleting the table which pulled it in.
