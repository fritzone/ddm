Table instances

When working in an object orineted modelling project the table instances are being created from a table template (in the project tree right click on the table template, choose Instantiate). This screen can be used to create the startup values of the tables (ie, values that will be inserted in the tables right after the creation. Warning! This screen is quite experimental in this version, so be careful when using it. There is the possibilty to import values from CSV files, and also you can save the values for further reference into CSV files. These values will be appended in the table generation SQL script in form of INSERT INTO ... lines of code.

With the button you insert a new row in the table, with the button you remove the currently selected row from the table. When you are done with inserting the data press the button to validate the data that was entered by you, and pressthe "Save" button to save the data. Basic validation is done at this, there are some basic error checkings, such as do not enter a string value for a numeric field, but the more complex validations are reserverd for future releases of the application.
