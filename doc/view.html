Views

A database View can be imagined as a stored query which upon invocation produces a result set that mimics the behavior of a table. Views are created using standard "SELECT" statements based on already existing tables or other expressions.

DDM offers support for two kind of view creation mechanisms:

  1. Manual view creation
  2. View creation with a query builder

Manual views

When choosing to create a view manually you will need to type in the entire SQL query for the creation of the view. DDM has no control over the query, the SQL will be simply stored in the save file as plain text and deployed in the database as you have typed in.This method of creating a view can be used by professionals and also beginner SQL developers if they feel confident enough. An example of an SQL for creating a view is (please feel free to copy/paste this in the SQL window in case you create views manually):

CREATE VIEW <MY_VIEW>
AS
SELECT * FROM <MY_TABLE>

When changing the name of the view from the query itself the name of the view in the name field and the project tree is updated, however when changin the name of the view from the name text field, the name in the SQL query takes precedence and it's not changed in the query, this allows you to give specific names for the views.

Query builder

DDM has a nice feature, called a query builder. This allows you to build the SQL query for a view using mostly the mouse, and without having to worry about mistyping a command or a table name. For a through introduction to the query builder please click here.
