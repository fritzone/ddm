Configuration

Code Generation

Use backticks
Specifies if you want to use backticks (`) in the SQL code generated. This is MySQL specific.

Upcase
If you want to generate UPCASE sql instead of lowercase SQL check this box. This will generate SQL commands, however it will not change the case-ness of the table and column names.

Primary key position
You can specify where to put the primary key declarations in the generated SQL.
The difference will be like:

"AfterColumnsDeclaration" will generate SQL like:
CREATE TABLE `TEST_TABLE`
(
`INDEX` INTEGER(10) DEFAULT 0,
`ANOTHER_INDEX` INTEGER(10) DEFAULT 0
,PRIMARY KEY (INDEX, ANOTHER_INDEX)
);
and "ColumnsDeclaration" will generate SQL like:
CREATE TABLE `TEST_TABLE`
(
`test_key` FLOAT PRIMARY KEY DEFAULT 0.0,
`few` BIT(1)
);
There is one exception: in case the table contains more than one primary key, regardless that the ColumnsDeclaration was specified, the code generator still will use the AfterColumnsDeclaration to generate valid SQLs.

Foreign Key Position
You can specify where to put the foreign keys in the generated SQL script.
"InTable" will put the FOREIGN KEY in the CREATE TABLE command
"AfterTable" will put the FOREIGN KEY in a separate ALTER TABLE command, outside the CREATE TABLE.
For the SQL prepared in the table details screen the foreign keys will not be shown, hoever they're present in the deployment SQL screen.

General

Allow foreign key propagation checkbox
Tells the application that when the user is specializing a table it should also take over the foreign keys in the new table. By default this is turned off.

Continuous validation
Tells the application to perform validation in every possible situation which can create database validation errors. If unchecked the validation will be executed only by pressing the Validate button on the main toolbar.
