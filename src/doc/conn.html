Connections

DDM supports the notion of "Connection" to a database. For you this basically means that you don't have to type in the database details every time you want to deploy something to a machine, but only once, when you create the connection, and from that point the connection is available for future use.

To create a connection click the button in the application main toolbar (or in the Connection tree) and fill in the required details. You can specify the database from the "Database Type" combobox.

Some controls, are common to all the databases. These are the following:

- "Name" of the connection - either give a proper name, or let DDM generate one for you
- "Auto Connect". Check the "Auto Connect" checkbox if you want to start this connection every time you start DDM.
- "Save Password" is a disabled checkbox reminding yo all the time that the password will be stored in plaintext in the connection configuration file.

The connection details vary for each of the different databases DDM supports.

For MySql they are the following

- "Host" - the host where this database is located. Type in the IP or the hostname
- The "Port" on which the mysql daemon is listening. The MySql default is 3306 but this might vary
- "User" / "Password" the required authentication data.

Now, click the "Connect" button.

The combo box of databases will be populated with the databases found at the given location. Or you can create a new database by clicking the "+" button.

For Sqlite the following controls are relevant:

- The "Version". DDM (via Qt) supports both version 2 and 3. Select the one you want.
- The "Database" entry is the file name where the Sqlite database is located. Either type in the name or use the "Browse" (yellow folder) button to open an existing one.

For CUBRID the following controls are meaningful:

- "Host" - the host where this database is located. Type in the IP or the hostname
- The "Port" on which the mysql daemon is listening. The CUBRID default port is 30000 but this might vary depending on the installation
- "User" / "Password" the required authentication data.
- In the "Database" field type in the name of the database and press "Test Connection" in order to see if it is valid or not. By pressing this button you will also get a list of available databases on that server.

Once you have populated all the required database fields you can save it by pressing the "Ok" button. From this point your connection is remembered for future use. You have all your connections listed in the Conenction tree (accessible from the "View" menu) where you can either test the connection or delete or modify them.
