Procedures and Functions


Stored procedures and functions are statements written in SQL language and stored in a database for later execution. For more details about the syntax please consult the documentation of your database. Stored procedures are available only for MySql and CUBRID projects, since Sqlite has no support for them.

DDM has support for two kind of Stored Method creation.

1. Guided method creation
2. Plain SQL method cration

Guided Method creation

The guided method creation feature allows you to specify elements of your method (such as the name, parameters name, type, description, etc...) in a controlled manner, and this data is used to check the validaty of the method, and also to generate the required documentation and the SQL code. When you use this way of creating a stored method the application will auto generate the first line of the SQL command (for MySql) or the entire SQL command (for CUBRID) and these line will be disabled in the SQL editor.

Since CUBRID supports Java based stored methods we reccomend reading the documentation if you don't understand the functionality.

Plain Sql Method creation

With DDM you can create a stored procedure/function using the SQL syntax of your database using the DDM text editor which tries to offer some limited code completion and other features to make your life easier.

We encourage you to add descriptive comments for your methods, in order for the automatic documentation generation tool to be able to generate documentation. The comments should be standard SQL comments (ie, starting with "-- ") and the following javadoc-like keywords are supported:

@brief - this should be the first line in your comment, please specify a one line description of the method
@param - for each of your parameters specify the name and a one line description.
@return - This is used only for functions, specifies the return type of the function
@desc - This should be the last keyword in the comment, starts a multi line description of your methods. All the lines in the comment after this will go to the description of the method.

If a line does not contain a create method keyword it is assumed that it is just description, and it will be used in the desription part of the documentation.
The following is an example on how to write a documentation:

-- @brief a procedure to test the documentation generator
-- @param p1 The parameter to the procedure
-- @desc This procedure just does nothing
CREATE PROCEDURE proc(p1 VARCHAR(255))
BEGIN
END

In case you did not provide a description for all the parameters, or a brief description the application will generate a TODO message for you. This is not customizable, we believe that the quality of the product depends on the documentation too, so please provide it to your users.
