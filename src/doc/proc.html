Procedures and Functions

Stored procedures and functions are statements written in SQL language and stored in a database for later execution. For more details about the syntax please consult the documentation of your database.

With DDM you can create a stored procedure/function using the SQL syntax of your database using the DDM text editor which tries to offer some limited code completion and other features to make your life easier.

We encourage you to add descriptive comments for your methods, in order for the automatic documentation generation tool to be able to generate documentation. The comments should be standard SQL comments (ie, starting with "-- " sinc the ) and the following javadoc-like keywords are supported:

@brief - this should be the first line in your comment, please specify a one line description of the method
@param - for each of your parameters specify the name and a one line description.
@return - This is used only for functions, specifies the return type of the function
@desc - This should be the last keyword in the comment, starts a multi line description of your methods. All the lines in the comment after this will go to the description of the method.

If a line does not contain a keyword it is assumed that it is just description, and it will be used in the desription part of the documentation.
The following is an example on how to write a documentation:

-- @brief a procedure to test the documentation generator
-- @param p1 The parameter to the procedure
-- @desc This procedure just does nothing
CREATE PROCEDURE proc(p1 VARCHAR(255))
BEGIN
END

In case you did not provide a description for all the parameters, or a brief description the application will generate a TODO message for you. This is not customizable, we believe that the quality of the product depends on the documentation too, so please provide it to your users.
