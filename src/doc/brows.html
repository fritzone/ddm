Browsing a connection

After you have created your connections you can browse them. To do this, either

1. Right click the connection in the connection tree and select "Refresh". This will connect to the database and retrieev all the elements found in it.

2. Double click the connection in the connection tree, this will automatically connect the connection and starts browsing it.

After these steps now you are able to see the tables, views, stored methods and triggers of the database represented by the selected connection. If you don't have a solution you just can browse the data stored in the table. For this, just double click the table (or right click and choose "Browse table data"). The tool offers a read-only view to the databse objects, however there is the possibilty for you to run scripts on the database.

If you have a solution you can import a table from the database into your solution, without the need to do a full reverse engineering of the database and you also have the possibilty to add tables from different databases to your solution, and deploy them later. At this stage it is possible to import only tables into the solution.

Other options regarding connections

When you right click a connection you will get a menu where the followings are the top level options you can choose from:

Run Query

By selecting this option an SQL editor window bound to the database specified in this connection will open. In this window you can type in SQL queries and execute them using Ctrl+Enter. Selecting this option automatically connects the database, starts browsing it and shows the content of the database. When you execute a data definition statement the connection tree is updated with the newly created object (or if you drop something the entry will be removed from the tree).

Test Connection

Selecting this option verifies whether the connection to the database is possible or not. This option does not start browsing the connection (ie: does not populate the tables, views, etc... items in the tree entry). If browsing the database connection is possible the icon is updated with a green signaller, otherwise it gets updated with a red one.

Edit Connection

Will open the window to modify the properties of the connection. This option does not perform operations on the database.

Delete

Choosing this option will remove the connection from the list of available connections, this does not affect the physical database.

Refresh

Choosing this option will refresh the tree objects of the database (ie: clear the tree, connect to the database, fetch all the items again and populate the tree again).

The "New..." top level menu entry is available for all databases. Contains:

New Table

You can create a new table in this connection.

The "Database" top level menu item with the following entries is visible only for MySql databases:

Recreate

Choosing this option will execute a "DROP DATABASE" and "CREATE DATABASE" script for the database represented by the given connection. This will remove all the objects from the database!

Drop

Choosing this option will execute a "DROP DATABASE" on the given connection.
