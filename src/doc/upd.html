Update management in Database Deployment Manager

Once you have more than one version there is the possibilty of generating an update script from a source version to target version. The update script should be injected into databases which have a deployment of the source version.

This is the procedure of executing a proper update:

1. Click the "Update" button (Orange star) from the main toolbar
2. Select the required "From" and "To" versions. From version 0.1k due to possible inconsistencies It is not possible to generate "downgrade" scripts.
3. Press the "Inject" button on the toolbar of the Update window. This will open the inject SQL dialog where you can select the destination database.

The followings are good to know before updating a database:

- if a View has the "Can replace" check box checked this change is ignored.
- if there are changes in the foreign keyness of the table (and the destination database supports this feature) the foreign keys are dropped, the table update scripts are run and the foreign keys are recreated.
- the changes of the startup values for the table instances are done based on the primary keys of the table. For example if you change one of the primary keys in a row of a table which has two columns designated as primary keys this will be interpreted as deletinga row, inserting a new one.
- deleting an object (table, table instance, etc...) and the creating another one with the same name is considered a drop/create operation.
- Sqlite does not support a wide range of alter table commands, so the only things that work are renaming of a table and addition of a column.
