Documentation

Database Deployment Manager offers the possibility to generate documentation for your project which is based on the description you have provided for the database objects. The following objects generate an entry in the documentation of the project:

 ∙ The Solution itself: The text you provide in the description for the solution is used in the documentation
 ∙ The Tables:
   ∘ The description of the table
   ∘ The description of the columns
   ∘ The indexes are presented
 ∙ The functions and procedures
  ∘ The documentation provided before the function code will be parsed and meaningful information extracted. See this for writing proper function documentation
 ∙ The triggers
  ∘ The description of the triggers

Stylesheets

Once you have loaded the documentation, you can change the style of the document using CSS style sheets from the DDM gui, in the combo box on the top. The default HTML elements can be styled using your own style, and in addendum the documentation generator uses the following style names to add more specialized styles:

.classProjectDescription - This is used to display the description of the project itself

.classDescription - This is used to display the description text of the database elements

.classTableHeader - This is used to display the text in the table headers (for columns, etc...). The background-color should match your "th" style from your CSS otherwise you'll get funny results.

.classTODO - This is used to display the TODO elements identified by DDM

.classTableText - This is used to display the text in the tables

.classCode - This is used to display code sequences (SQL for now)
