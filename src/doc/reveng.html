Reverse engineering

Reverse engineering a database means to take all the objects found in it and create a set of DDM objects (tables, views, etc...) based on the objects that are defined in that database. DDM supports the reverse engineering of all three databases it supports, with small limitations as imposed by the database engines.

When you create a new project you will be offered the option to reverse engineer a database. Firstly you will need to select the database engine, and then fill up the connection details to the database you want to reverse engineer. The tool will offer you the list of database objects you can reverse engineer.
